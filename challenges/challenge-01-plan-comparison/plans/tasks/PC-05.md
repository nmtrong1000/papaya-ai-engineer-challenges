# PC-05. Implement best-value highlighting per row

## Requirement

In each comparable row, the cell(s) with the best value are visually highlighted with a distinct background or text colour.

## Approach

Create `src/utils/bestValue.ts` that exports a `getBestPlanIndex(row: RowKey, plans: Plans): number[]` function. "Best" means: highest for limits and annual cap, lowest for copay and waiting period, highest for benefit sub-fields (treating `-1` as Infinity). The function returns an array of plan indices so ties are supported. In `ComparisonTable.tsx`, call `getBestPlanIndex` per row and apply a Tailwind highlight class (e.g. `bg-yellow-50 font-semibold`) to the winning cell(s). Non-benefit rows (premium, annual limit, copay, waiting period) use simple numeric comparison; benefit rows compare the primary numeric field for each type.

## Execution Steps

- [ ] Create `src/utils/bestValue.ts`
- [ ] Define a `RowKey` type covering all comparable rows
- [ ] Implement comparison logic for each row key; treat `-1` as `Infinity`
- [ ] Export `getBestPlanIndex(row, plans): number[]`
- [ ] In `ComparisonTable.tsx`, call `getBestPlanIndex` for each row and conditionally apply highlight Tailwind classes to winning cells
- [ ] Confirm visually in the dev server
