# PC-03. Build plan card & comparison table — scalar fields

## Requirement

A `PlanCard` component and a `ComparisonTable` component exist: the table receives `plans`, passes each plan to a card, and the card renders the plan name, monthly premium, annual limit, copay %, and waiting period — completely and correctly, with no benefit rows yet.

## Approach

Create `src/components/PlanCard.tsx` that accepts a `Plan` prop and renders only the five scalar fields. Create `src/components/ComparisonTable.tsx` that accepts `plans: Plans`, maps over them, and renders one `PlanCard` per plan inside a `flex flex-col gap-4 md:flex-row` container — this single rule handles both desktop (horizontal) and mobile (vertical) layout. Mount `ComparisonTable` in `App.tsx`. Benefit rows are out of scope for this task.

## Execution Steps

- [ ] Create `src/components/PlanCard.tsx` with a `plan: Plan` prop
- [ ] Render in the card: plan name as a heading, monthly premium, annual limit, copay % (show "None" when 0), waiting period in days (show "None" when 0)
- [ ] Format all numeric values with `toLocaleString()` where appropriate
- [ ] Style `PlanCard` with Tailwind: `flex-1 border rounded-xl p-6 shadow-sm bg-white`
- [ ] Create `src/components/ComparisonTable.tsx` accepting `plans: Plans`; render `<PlanCard plan={plan} key={plan.name} />` inside `flex flex-col gap-4 md:flex-row`
- [ ] Replace `App.tsx` body with `<ComparisonTable plans={plans} />`; add `bg-gray-50 min-h-screen p-6` to the wrapper
- [ ] Run dev server and verify
