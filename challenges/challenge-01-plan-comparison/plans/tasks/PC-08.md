# PC-08. Polish visuals & accessibility

## Requirement

The page meets WCAG AA contrast minimums, uses semantic HTML, and looks professional enough for a health insurance product page.

## Approach

Audit and tighten Tailwind classes across the table and cards: ensure highlight colours pass 4.5:1 contrast, add `scope="col"` / `scope="row"` to `<th>` elements, add an `aria-label` to the table, and add a page `<header>` with a title and subtitle. Replace any bare ✕ text with an SVG icon that has `aria-hidden="true"` and a visually-hidden screen-reader label. Add plan `highlights` as a bullet list beneath each plan header. Final check: no `console.error` or warning on load.

## Execution Steps

- [ ] Add `<header>` section above the table with page title ("Insurance Plan Comparison") and a one-line subtitle
- [ ] Add `scope="col"` to plan name `<th>` cells and `scope="row"` to label `<th>` cells
- [ ] Add `aria-label="Insurance plan comparison table"` to `<table>`
- [ ] Render `plan.highlights` as a `<ul>` in each plan header cell (table) and card header (mobile)
- [ ] Replace ✕ text with an SVG cross icon (`aria-hidden="true"`) + `<span className="sr-only">Not included</span>`
- [ ] Verify highlight colour contrast in DevTools (Accessibility panel)
- [ ] Confirm zero console errors on load

## How to Test

npm run dev

- Page has a visible title
- Plan highlights bullet list appears under each plan name
- "Not included" cells have an icon + visually-hidden text (inspect in DevTools)
