# PC-09. Deploy to Vercel & smoke test

## Requirement

The app is live on a public Vercel URL, the GitHub repository is public, and the README documents the stack and how to run locally.

## Approach

Initialise a git repo in the challenge directory (or repo root), push to GitHub, then connect to Vercel via the dashboard or CLI. Vercel auto-detects Vite and sets the correct build command (`npm run build`) and output directory (`dist`). After deploy, run a smoke test against the live URL. Write the README in the challenge directory.

## Execution Steps

- [ ] `git init` in `challenges/challenge-01-plan-comparison/app/` (or confirm it is tracked by the parent repo)
- [ ] Add `.gitignore` entries for `node_modules/`, `dist/`
- [ ] Commit all files: `git add . && git commit -m "feat: insurance plan comparison page"`
- [ ] Push to a new public GitHub repository
- [ ] Deploy via Vercel: `npx vercel --prod` or connect the repo in the Vercel dashboard
- [ ] Confirm build succeeds and live URL is accessible
- [ ] Create `challenges/challenge-01-plan-comparison/README.md` with: what was built, stack, `npm run dev` instructions, and the live URL
- [ ] Smoke test the live URL: table renders, Recommended badge visible, mobile view works

## How to Test

```bash
# Local build check
cd challenges/challenge-01-plan-comparison/app
npm run build && npm run preview
# Open http://localhost:4173 — full table visible
```

Live URL smoke test checklist:
- [ ] Page loads without errors
- [ ] Three plan columns visible on desktop
- [ ] "Recommended" badge on Gold
- [ ] Best-value cells highlighted
- [ ] Mobile (375 px) shows stacked cards
- [ ] No console errors
