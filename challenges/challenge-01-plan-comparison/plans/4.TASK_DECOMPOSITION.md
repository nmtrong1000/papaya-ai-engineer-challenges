## Rationale

- **01 before 02:** Toolchain must exist before data files can use TypeScript path aliases.
- **02 before 03:** The table component imports the typed data; the shape must be finalised first.
- **03 before 04:** The skeleton establishes the column/row structure; benefit rendering fills it in.
- **04 before 05–06:** Highlighting and badge logic operate on the rendered benefit values; they depend on the row structure being stable.
- **05–06 before 07:** Logic is verified on desktop before the layout is rearranged for mobile.
- **07 before 08:** Responsive structure is confirmed before adding decorative polish.
- **08 before 09:** Final visual state is locked before capturing the live URL for submission.
