# Phase 1 — Requirement Analysis

## 1.1 Problem Statement

Insurance companies sell multiple plan tiers with different benefit structures, pricing, and coverage limits. Prospective members struggle to evaluate these plans without a clear side-by-side view. This challenge requires building a web page that presents three plans — Bronze, Silver, and Gold — in a comparison table so a non-technical user can immediately see what each plan covers, where the gaps are, and which plan offers the best overall value for money.

## 1.2 Functional Requirements

- [ ] FR-01: Display all three plans (Bronze, Silver, Gold) in a side-by-side comparison table
- [ ] FR-02: Show monthly premium for each plan
- [ ] FR-03: Show annual limit for each plan
- [ ] FR-04: Show outpatient benefit details (limit per visit, visits per year) — display "Unlimited" when value is -1
- [ ] FR-05: Show inpatient benefit details (limit per day, days per year) — display "Unlimited" when value is -1
- [ ] FR-06: Show dental benefit with annual limit, or a visual "Not included" indicator when null
- [ ] FR-07: Show maternity benefit with per-pregnancy limit, or a visual "Not included" indicator when null
- [ ] FR-08: Show copay percentage for each plan
- [ ] FR-09: Show waiting period in days for each plan
- [ ] FR-10: Show plan highlights as a bullet list for each plan
- [ ] FR-11: Highlight the best value in each comparable row (e.g., highest limit, lowest copay, lowest waiting period)
- [ ] FR-12: Display a "Recommended" badge on the plan with the best value-for-money ratio
- [ ] FR-13: Responsive layout — columns stack vertically on mobile viewports
- [ ] FR-14: Visual indicators (icons or styled text) to distinguish included vs not-included benefits

## 1.3 Non-Functional Requirements

- **Performance:** Page must load and render without JavaScript build errors; no runtime exceptions
- **Accessibility:** Semantic HTML table or equivalent ARIA structure so screen readers can traverse columns; sufficient color contrast (WCAG AA minimum)
- **Browser support:** Modern evergreen browsers (Chrome, Firefox, Safari, Edge)
- **Responsiveness:** Breakpoint at ≤768 px — table must be usable on mobile without horizontal scroll or truncation
- **Design quality:** Professional, clean aesthetic suitable for a health insurance product page
- **No external data fetching:** All plan data is embedded in the codebase (static JSON); no API calls required

## 1.4 Out of Scope

- User authentication or login
- Plan purchase / checkout flow
- Dynamic plan data loaded from a backend API
- Filtering or sorting of plans
- Quote generation or premium calculator
- Multi-language / i18n support
- Dark mode
- Animations beyond simple hover states
- Any plan beyond the three provided (Bronze, Silver, Gold)

## 1.5 Definition of Done

- [ ] All 14 functional requirements are implemented and visible in the browser
- [ ] "Recommended" badge appears on exactly one plan (Gold, per value-for-money logic)
- [ ] Best-value cells are visually highlighted in each row
- [ ] Null benefits render as a clear "Not included" indicator (not blank or "null")
- [ ] `-1` values render as "Unlimited" (not "-1")
- [ ] Layout is responsive: tested at 375 px (mobile) and 1280 px (desktop)
- [ ] No console errors on the golden path (page load, resize)
- [ ] Code is organized and readable — data separated from presentation
- [ ] App is deployed to a free hosting service and the live URL is accessible
- [ ] GitHub repository is public with a README explaining the stack and how to run locally
